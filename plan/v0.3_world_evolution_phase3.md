# v0.3 — 世界演化语料 + 稳定性/扰动检验（完整 Phase 3 真比较器）

> **目标**：执行协议 §5 真正的比较器——不再是"谁被升层"（v0.1/v0.2 只测这个），而是**"存活记忆 vs 演化中外部世界后期发展的一致性"**，即测量"稳定 vs 易被破坏"。这是约束场理论核心后果（Internal Selection 产生易被破坏的不稳定结构）**唯一能被经验检验**的路径。
>
> **依赖**：v0.2（两道闸都可切换且能真实触发）。

---

## 1. 为什么需要这一版

v0.1 Phase 3 pilot 与 v0.2 都只测了"哪些记忆被升层"——这是**机制分歧**的可观测足迹，但**不是理论预测的后果**。理论预测的后果是：

- External Selection 产生的结构**能经受演化中的外部世界**（稳定）。
- Internal Selection 产生的结构**易被破坏**（不稳定，因选择压力可被自身消除）。

[`docs/EXPERIMENT_REPORT.md §④`](../docs/EXPERIMENT_REPORT.md) 与 [`docs/RELEASE_v0.1.md`](../docs/RELEASE_v0.1.md) 已诚实声明"稳定 vs 易被破坏尚未测量，协议 §5 真比较器未执行"。v0.3 就是来执行它的。

## 2. 实验设计：世界演化语料

### 2.1 两段式语料结构

| 段      | 名称                      | 作用                                                            | 性质                                                                                   |
| ------- | ------------------------- | --------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Phase A | 形成期（Formation）       | 让两策略各自形成 working/consolidated/core 结构                 | 沿用 v0.2 语料设计（复述累积 → 升层 → 重组 → 升慢层）                                  |
| Phase B | 演化期（World Evolution） | 外部世界发生变化：引入与既有 core 记忆**矛盾/更新**的新外部输入 | 关键：这些输入是**实验者未控制的自然演化**（如用户改了偏好、事实更新），仍守反身性红线 |

### 2.2 关键变量：Phase B 的"世界演化"必须是外部驱动的

- Phase B 不能是系统自问自答、不能是诱导确认。
- 例：Phase A 形成 "用户喜欢羽毛球"（core）；Phase B 引入 "用户最近膝盖受伤，医生建议停止羽毛球，改游泳"——这是**外部世界的真实演化**，与既有 core 记忆产生张力。
- 这正是协议 §3 反身性防护的核心检验点：选择压力必须来自不可控外部输入。

## 3. 稳定性度量（核心新增）

### 3.1 慢层记忆的"存活/失效"标签

对每条 core 记忆，在 Phase B 后标注：

| 标签                    | 定义                                                                                    |
| ----------------------- | --------------------------------------------------------------------------------------- |
| **稳定（stable）**      | core 记忆与 Phase B 演化后的外部世界**仍一致**，或被自然更新而非推翻                    |
| **易被破坏（brittle）** | core 记忆与 Phase B 演化**矛盾**且未被系统修正（仍作为"事实"留在 core，但已与现实不符） |

### 3.2 候选度量（PI 选定后写入协议 v5）

- **一致性率**：`stable_core / total_core`。预测 persistence > frequency。
- **矛盾滞留率**：Phase B 引入矛盾后，core 中与新世界矛盾的记忆占比。预测 frequency > persistence（纯内部访问升层者更易与现实脱节）。
- **结构响应度**：Phase B 矛盾输入是否触发了 reorg（合并/矛盾标记）。预测 persistence 模式更可能触发结构更新（因 reorg 是其升层机制）；frequency 模式可能"无视"矛盾（因升层只看访问数）。

### 3.3 预测（可证伪）

- **若理论成立**：persistence 形成的 core 在 Phase B 后一致性更高、矛盾滞留更低；frequency 形成的 core 更易与现实脱节。
- **若证伪**：两策略的稳定性无差异，或 frequency 反而更稳——则"选择压力来源决定结构稳定性"的命题受挑战。

## 4. 对照设置

- 同一份两段式语料，仅切 `--strategy`，两道闸联动。
- Phase A 两策略提取集一致（验证）；Phase B 同样。
- 关键：Phase B 的"演化方向"对两策略**完全相同**（同一批外部输入），唯一变量仍是 Migration Rule。

---

## 5. 诚实边界（v0.3 须声明）

- 仍是单一手工构造语料，n=1。Phase B 的"演化"由实验者设计，存在构造偏误（演化方向预先决定哪些 core 会失效）。**消偏留 v0.4 LoCoMo**。
- "稳定/易被破坏"标签的判定可能需要 LLM 辅助标注（core 记忆 vs Phase B 输入是否一致）——引入新的 LLM 判定自由度，须冻结 Prompt、temperature=0，并声明。
- 统计推断仍弱（单语料无置信区间）；v0.4 多语料才补。
