## 1. 反事实重要性（Counterfactual Importance）

### 动机

[`docs/Memory_Architecture_设计文档_MVP.md §十三.2`](../docs/Memory_Architecture_设计文档_MVP.md) 与 [`docs/01_PROTOCOL_v3.md §1.4`](../docs/01_PROTOCOL_v3.md) 都声明：当前 `local_reorganization_trigger` 基于 cosine ≥ 0.90 的**局部语义重叠**，只捕近重复重组，无法捕捉因果网络中的关键节点（A拥有X，X属于B，B能开启C——X 不与 A/C 语义相似但是关键节点）。

### 目标

把中→慢判据从"语义重叠触发重组"升级为"**删除该记忆后多少推理路径失效**"——即反事实重要性。这是 persistence-based 迁移机制的理论完整版。

### 方法（候选）

- 对每条中层记忆 m：暂时移除 m，跑一组推理任务，测量输出/检索结果的变化幅度。
- 变化大 = m 是结构关键节点 → 升慢层候选。
- 需要先有 Phase 2 Reply 闭环（见 §3）才能定义"推理路径"。

### 依赖

- Phase 2 Reply 闭环（§3）。
- 这是协议层大改（迁移判据重定义），须升协议版本。

## 2. 被动衰减（Passive Decay）

### 动机

[`ananke/models.py`](../../ananke/models.py) 已预留 `decay_coefficient` 与 `persistence_score_log` 字段。协议 v3 §1.3 明确"终身累积、不重置、不衰减（衰减属 v0.2）"。本项落实衰减。

### 设计

- 定期扫描：用当前时间与 `last_activated_at` 差值 × λ 扣减 persistence_score。
- 关键问题（[`docs/Memory_Architecture_设计文档_MVP.md §十三.1`](../docs/Memory_Architecture_设计文档_MVP.md)）：不同层的 λ 应差多少？理论要求沉淀速率分离，故 λ_working > λ_consolidated > λ_core（快层衰减最快）。
- ⚠️ 衰减与"终身累积不重置"是不同的实现映射，须作为新映射单独冻结协议版本。

## 3. Phase 2 Reply 闭环

### 动机

[`docs/00_THEORY.md §7`](../docs/00_THEORY.md) Phase 2 = Implementation → Dynamics：闭环是否真产生可观测的长期行为（**慢层持续塑形快层推理**）。当前 [`ananke/pipeline.py`](../../ananke/pipeline.py) 的 `retrieve()` 只做优先级拼接，**未让检索影响输出**——"慢层塑形快层"的最后一公里未闭合。

### 目标

接 Reply：用检索到的三层记忆（慢>中>快）构造 LLM 回复，使慢层结构真实影响快层推理。

### 纪律红线（防漂移）

- [`docs/03_RESEARCH_LOG.md`](../docs/03_RESEARCH_LOG.md) 已记：曾把"检索增强回复"当演示/体验推——那是能力视角漂移。
- Phase 2 Reply 的**唯一理论目的** = 让"慢层塑形快层"可观测，**不是**提升回复质量。Reply 质量不进 Evidence（[`docs/00_THEORY.md §5`](../docs/00_THEORY.md)）。
- 验证对象：慢层记忆是否稳定改变快层推理方向（Dynamics），而非回复好坏。

### 依赖

- v0.2（慢层有真实内容可塑形）。

## 4. Memory Identity 协议化

### 动机

[`docs/01_PROTOCOL_v3.md §1.7`](../docs/01_PROTOCOL_v3.md) 已显式命名"记忆同一性阈值 0.80"，但仍是隐含统一。GPT 与 GLM 两次独立评审收敛到"应把它提升为协议一级概念"。

### 目标

把 0.80 从"EV/dedup 共用的工程常量"提升为协议级 `Memory Identity Threshold`，统一 EV / dedup / Memory Unit 边界定义。跨 LLM 对比时才生效（[`docs/RELEASE_v0.1.md`](../docs/RELEASE_v0.1.md) OUT 项）。

## 5. 慢层主动塑形

### 动机

[`docs/Memory_Architecture_设计文档_MVP.md §十三.7`](../docs/Memory_Architecture_设计文档_MVP.md)：MVP 慢层只被动参与检索。理论完整版中，慢层应主动影响下层（如改变快/中层的激活阈值）——对应约束场理论"慢层约束场塑形快层可能性空间"。

### 状态

远期，依赖反事实重要性 + Phase 2 闭环。当前不设计。

## 6. 跨 LLM 一致性（LLM Invariance）

### 动机

当前所有结论条件于单一 LLM（Gemini）。理论不依赖具体 LLM，但实现映射可能依赖。

### 方法

同协议、同语料、换 LLM（DeepSeek / GPT / Claude）重跑，检验动力学轨迹方向是否一致。曾被 GPT 自判为漂移挂起（[`docs/03_RESEARCH_LOG.md`](../docs/03_RESEARCH_LOG.md)），Phase 3 后再启。
