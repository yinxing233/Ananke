# v0.2 — 中→慢双策略可切换 + 真实触发

> **目标**：让 persistence（External Selection）与 frequency（Internal Selection）的差异，在**第二道迁移闸（consolidated→core）**上可观测。这是 v0.1 之后最优先的一步——因为慢层是"最稳定结构"，理论预测的 persistence/frequency 分歧本应在这里最显著，而当前这道闸既不可切换、又从未真实触发。
>
> **依赖**：协议须升 v4（新增 consolidated→core 的策略化判定函数），按原则 C 不得静默改 v3。

---

## 1. 现状缺口（基于代码与 v0.1 证据）

### 1.1 中→慢代码在，但从未真实触发

- [`ananke/migration.py:58`](../../ananke/migration.py) [`promote_consolidated_memories()`](../../ananke/migration.py) 已实现并被 [`ananke/pipeline.py:53`](../../ananke/pipeline.py) 调用。
- 但 [`docs/RELEASE_v0.1.md`](../../docs/RELEASE_v0.1.md) OUT 清单明列"中→慢迁移在真实 LLM 下的触发"未做；[`docs/03_RESEARCH_LOG.md`](../../docs/03_RESEARCH_LOG.md) 记录 Mochi 矛盾句 `local_reorganization=0`。

### 1.2 根因一：reorg trigger 无法跨轮累积

[`ananke/reorganization.py:14`](../../ananke/reorganization.py) [`check_local_reorganization()`](../../ananke/reorganization.py) **只在记忆首次晋升到中层时调用一次**（[`pipeline.py:49`](../../ananke/pipeline.py)），且只给 `new_memory` 计 [`local_reorganization_trigger`](../../ananke/models.py)。

- 一条记忆若首次检查时 trigger < 2，之后**永无机会再累积**。
- 要在单次检查内拿到 trigger ≥ 2，必须同时撞上 ≥2 条已在中层、且 cosine ≥ 0.90、且 LLM 判合并/矛盾的旧记忆——条件极窄。
- 这与设计文档 §4.2"这条信息至少两次导致了中层已有知识的局部重组"的**跨时间累积**语义不符。

### 1.3 根因二：0.90 阈值只捕近重复，不捕逻辑矛盾

[`Config.REORG_SIMILARITY_THRESHOLD = 0.9`](../../ananke/config.py) 只在**近重复措辞**间触发 LLM 三选一。Mochi 的 "playful" vs "doesn't like touched" 是逻辑矛盾但表述不同，cosine < 0.90，永远进不了 reorg 判定（[`docs/03_RESEARCH_LOG.md`](../../docs/03_RESEARCH_LOG.md) 已记为"设计属性，是否拓宽是 Phase 2/3 议题"）。

### 1.4 根因三（最关键）：中→慢不可策略切换

[`promote_consolidated_memories()`](../../ananke/migration.py) 不接收 strategy 参数。persistence 与 frequency 两次跑用**完全相同**的 `local_reorganization_trigger ≥ 2`。**即便它触发，也无法展现 persistence vs frequency 差异**——对照实验的"唯一变量 = Migration Rule"在第二道闸上失效。这是 v0.1 Phase 3 pilot 只能停在快→中的结构性原因。

> **安抚性推论（v0.1 结论不受影响）**：v0.1 真实运行中**没有任何记忆走到第二道闸**（reorg 从未触发，[`docs/03_RESEARCH_LOG.md`](../docs/03_RESEARCH_LOG.md) 记 `local_reorganization=0`）。因此"两策略共用中→慢规则"**没有产生过任何观测效应**——v0.1 的 Phase 3 pilot 结论（分歧落在快→中那一刀）成立且不受本缺口影响。本缺口只在 v0.2 让第二道闸真正运行后才成为问题。

### 1.5 根因四（结构性死结，最致命）：reorg 触发窗口被 dedup 掏空

[`Config`](../../ananke/config.py) 中 `REORG_SIMILARITY_THRESHOLD = 0.90` **大于** `DEDUP_SIMILARITY_THRESHOLD = 0.80`。而 [`ananke/pipeline.py`](../../ananke/pipeline.py) 的写入前去重在 reorg 之前执行：

- 任何 cosine ≥ 0.90（够格触发 reorg）的候选提取物，**必然 ≥ 0.80**，会在写入前被 dedup 拦截、永不成为记忆、永不晋升、**永远触发不了 reorg**。
- 即 reorg 的触发窗口 `[0.90, 1.0]` 是 dedup 拦截区 `[0.80, 1.0]` 的**真子集**——reorg 对"新写入记忆 vs 既有记忆"这条路径**数学上为空集**。
- 这不是语料写得好不好的问题，是阈值排序的结构性死结。（[`tools/dev_simulate.py`](../../tools/dev_simulate.py) 靠把 reorg 降到 0.70 绕过，但那是开发演示，真实实验不能这么绕。）

> 唯一不被死结封死的窄缝：reorg 检查的是"新晋升到中层的记忆 vs 既有中层记忆"。若两条记忆在不同时刻写入（写入时对方还不在 store 里，故各自未被 dedup），后来双双晋升中层，它们之间仍可能有 cosine ≥ 0.90。但这要求**两条本就高度近似的记忆错时写入**——在真实 LLM 提取下极难满足（近义输入会同时被提成近似提取物并同时撞 dedup）。因此死结在实践上仍基本封死 reorg。

**这个死结必须在协议 v4 里先解开，才能谈中→慢真实触发**（见 §2.3 PI 决策点）。

## 2. 设计：把第二道闸也变成"选择压力来源"的对照

### 2.1 理论框架（写入协议 v4）

Phase 3 比较的是**选择压力的来源**（Internal vs External Selection），跨**整个系统**。两道闸都应反映该区别。

> **"唯一变量 = Migration Rule" 声明升级**：v0.1 该声明只作用于快→中一道闸。v0.2 实施后必须升级为"**两道闸联动切换**"——`--strategy` 同时切 working→consolidated 与 consolidated→core 两道闸，保证对照实验的唯一变量仍是选择压力来源，而非"哪道闸切了"。若未来需要单独研究某道闸，再拆 `--working-strategy` / `--core-strategy`，但 MVP 对照实验默认联动。

| 闸    | External Selection（persistence）                                                    | Internal Selection（frequency）                               |
| ----- | ------------------------------------------------------------------------------------ | ------------------------------------------------------------- |
| 快→中 | `EV×1.0 + IA×1/e ≥ 3.0`（环境证据加权）                                              | `total_activation ≥ 3`（内部访问计数，不区分来源）            |
| 中→慢 | `local_reorganization_trigger ≥ 2`（**结构性重组**，重组事件由外部到达的新记忆驱动） | `total_activation ≥ T_core`（**访问频率**，忽略是否重组结构） |

**理论自洽性检查**：

- External Selection 的中→慢判据是**结构性**的——一条记忆晋升慢层是因为它**反复改组了中层已有知识**，而重组事件由外部输入（新记忆到达）触发，选择压力源自环境。这满足原则 A"不同时间尺度有不同更新机制"（中→慢是结构型，区别于快→中的重复激活型）。
- Internal Selection 的中→慢判据退化为**访问计数**——与快→中同构，正是理论预测的"内部自循环退化"：慢层结构由系统自身检索频率决定，可被自身消除。这是**刻意的退化基线**，不违反原则 A，而是原则 A 的反例对照。
- ⚠️ **PI 决策点**：frequency 在中→慢退化为访问计数，是否需要在协议中显式声明"此为退化基线、非理论主张"？建议声明，避免被读成"理论认为访问计数是合理的慢层形成机制"。

### 2.2 代码改动

**新增 [`ananke/promotion.py`](../../ananke/promotion.py) 的对偶层**：

```
ConsolidatedPromotionStrategy(ABC):
    name: str
    threshold / score(memory) / should_promote(memory)

PersistenceCoreStrategy:   score = local_reorganization_trigger ; threshold = LOCAL_REORG_THRESHOLD (2)
FrequencyCoreStrategy:     score = total_activation            ; threshold = FREQUENCY_CORE_THRESHOLD (新常量，待标定)
promotion_strategy_for_consolidated() 工厂
```

**改 [`ananke/migration.py:58`](../../ananke/migration.py)**：`promote_consolidated_memories(memory_store, event_logger, strategy=None)`，`strategy` 默认从 `Config.CONSOLIDATED_PROMOTION_STRATEGY` 取；日志记 `migration_strategy` + 双分数。

**改 [`ananke/config.py`](../../ananke/config.py)**：新增 `CONSOLIDATED_PROMOTION_STRATEGY = os.getenv(..., "persistence")` 与 `FREQUENCY_CORE_THRESHOLD`。

**改 [`ananke/pipeline.py`](../../ananke/pipeline.py)**：把 consolidated promotion 也注入策略；`run_corpus.py` 的 `--strategy` 同时切两道闸（或拆成 `--working-strategy` / `--core-strategy`，但 MVP 建议单开关联动，保证"唯一变量"语义）。

### 2.3 解死结 + 修累积缺陷（协议 v4 核心决策项）

§1.5 的死结必须先解，reorg 才可能触发；§1.2 的累积缺陷同时要修。这两件事可合并为**一个**设计决策。下面列出四个候选方向，**PI 在协议 v4 选定**（实施时再定，本计划不预判）：

- **方向一（调阈值开窗口）**：把 `DEDUP` 升到 0.95、`REORG` 降到 0.85，制造 `[0.85, 0.95)` 的窗口——句子够近能触发 reorg、又不够近到被判同一记忆。
  - 缺点：两个阈值都动，v0.1 的去重语义（"记忆同一性阈值 0.80"）被改写，需重新校准，且与 EV 阈值 0.80 的耦合关系破裂（[`docs/01_PROTOCOL_v3.md §1.7`](../docs/01_PROTOCOL_v3.md) 的 Memory Identity 命名将受冲击）。

- **方向二（每轮重跑 reorg）**：对中层记忆每轮重跑 reorg 检查。
  - 缺点：LLM 调用增多，**且单独用时仍受死结束缚**（新写入候选仍被 dedup 拦在门外）——除非配合方向一。

- **方案 A（晋升时双方计 trigger）**：reorg 仍在晋升时触发，但当新记忆 C 与既有 A 合并/矛盾时，**同时给 A 和 C 各 +1 trigger**。
  - 缺点：**仍受死结束缚**（C 必须先逃过 dedup 才能晋升到中层去触发 reorg）；且给 A 计数的语义与设计文档 §4.2"该记忆**导致**了重组（触发源）"有微妙偏移——A 在这里是"**被**重组（受体）"。

- **方向三（推荐：把 dedup 命中变成 reorg 入口）**：当一条新提取物因 cosine ≥ 0.80 被 dedup 拦截、**且匹配到的既有记忆在中层**时，不只是记一条 `memory_dedup_skip`——把这对文本送进 LLM 三选一；若判"矛盾"或"合并"，给**既有中层记忆 A** 的 `local_reorganization_trigger +1`。
  - 同时解决三个问题：
    1. **死结消失**——正是那些"近到被去重"的输入携带着重组信号；dedup 不再丢弃它，而是把它路由到 reorg。
    2. **跨轮累积自然成立**——A 每次被近重复输入撞上都可能 +1，不依赖"首次晋升单次检查"。
    3. **语义更忠实**——"外部世界反复送来与 A 高度重叠但有出入的信息"正是"A 是被持续检验的争议结构节点"的操作化，比方案 A"C 晋升时给 A 捎带 +1"更直接。
  - 代价：dedup 路径新增 LLM 调用，但**仅限匹配到中层记忆时**触发（匹配到快层记忆不调），量可控。
  - **语义统一**：方向三下 A 既是"被撞击的对象"也是"持续吸引外部检验的结构节点"——"导致重组"与"被重组"在 A 身上统一（设计文档 §4.2 的触发源语义可顺势修订为"被持续检验的结构节点"，协议 v4 写定义时写清）。

> **本计划默认按方向三推进**，但**须 PI 在协议 v4 批准**。方向三与方案 A 可叠加（方向三负责 dedup 路径的累积，晋升时 reorg 检查仍保留作窄缝补充）。

### 2.4 reorg 阈值是否拓宽

0.90 只捕近重复矛盾。v0.2 的目的是"让中→慢可触发 + 可对照"，**不是**"拓宽矛盾检测精度"。

- 若选**方向三**：reorg 的 cosine 门槛实际由 dedup 的 0.80 决定（dedup 命中即送 LLM），0.90 退居为"晋升时窄缝检查"的阈值——两者并存，语义不冲突。
- 若选**方向一**：reorg 阈值本身被下调到 0.85。
- 逻辑矛盾检测拓宽（捕"playful vs doesn't like touched"这类非近重复矛盾）归 v0.5（反事实重要性方向），**不在本版**。

## 3. 语料设计：让中→慢在真实 LLM 下触发

目标：在 persistence 模式下，至少 1 条记忆走完 working→consolidated→core 全链路。

### 3.1 语料结构（`corpus_phase2.txt`，建议 ~40–60 句）

> ⚠️ **语料设计依赖 §2.3 的方向选择**。下表默认按**方向三**写（dedup 命中即触发 reorg，候选输入因 ≥0.80 被拦但信号路由到 A）。若 PI 选方向一/二/A，措辞门槛随之调整。

| 段                 | 作用                          | 设计（方向三下）                                                                                                                                               |
| ------------------ | ----------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| S1 复述累积        | 让主事实 A 升中层             | 同一事实（如 "用户每周六打羽毛球"）6+ 句**近义复述**，触发 EV 累积 → persistence ≥ 3.0 → A 升中层                                                              |
| S2 让 A 成为中层   | 确保 A 在中层、可被后续重组撞 | （S1 已让 A 升中层；此段可放噪声或第二个独立事实，增加中层多样性）                                                                                             |
| S3 近重复矛盾/合并 | 触发 reorg，给 A 累积 trigger | 引入 2 句与 A **近重复（cosine ≥ 0.80，必触发 dedup）但合并/矛盾**的句子（如 "用户周六打羽毛球" vs "用户周六改打网球"）。dedup 拦截→送 LLM 三选一→A.trigger +1 |
| S4 再次重组        | A.trigger 达 2 → 升慢层       | 再引入 1–2 句近重复重组句 → A.trigger = 2 → consolidated→core                                                                                                  |
| S5 噪声            | 验证停快层                    | 无关事实若干                                                                                                                                                   |

### 3.2 关键约束

- **反身性红线**：所有句子是外部语料，非系统生成/诱导。
- **措辞门槛随方向而定（方向三下）**：S3/S4 只需保证与 A 的 cosine **≥ 0.80**（触发 dedup 命中即送 LLM）。若 PI 选方向一，则需 ≥ 0.85（调阈值后的 reorg 窗口）。这是 v0.2 的可控构造，**构造偏误须在报告中声明**（与 v0.1 同纪律），消偏留 v0.4 LoCoMo。
- **提取同语言**：英文语料，沿用 v3 提取 Prompt。

## 4. 实验设计：观察 persistence vs frequency 在中→慢的区别

### 4.1 对照设置（守住协议 §5 比较器）

- 同一份 `corpus_phase2.txt`，仅切 `--strategy persistence` / `--strategy frequency`，两道闸联动切换。
- 同 LLM、同嵌入、同提取/检索/reorg 逻辑、temperature=0。
- 验证两遍提取集一致（沿用 [`logs/_phase3_compare.py`](../../logs/_phase3_compare.py) 思路）→ 唯一变量 = Migration Rule（两道闸）。

### 4.2 可观测指标（中→慢这道闸）

| 指标                                                 | persistence 预测               | frequency 预测                 |
| ---------------------------------------------------- | ------------------------------ | ------------------------------ |
| 升慢层记忆数                                         | 少（仅结构性重组者）           | 多（高访问者即可）             |
| 升慢层记忆中 `local_reorganization_trigger > 0` 占比 | ~100%                          | 低（访问计数升层，与重组无关） |
| 升慢层记忆中 EV>0 占比                               | 高（重组由外部到达的记忆驱动） | 低（纯内部访问可升）           |
| 升慢层记忆是否改组过中层结构                         | 是                             | 不要求                         |

**分歧判据**：frequency 把"零结构重组、纯内部访问"的记忆推上慢层，而 persistence 不推——这正是 Internal Selection 在**最稳定层**的退化足迹，比快→中那一刀更接近理论核心。

### 4.3 诚实边界（v0.2 必须声明）

- 仍只测"谁被升层"，**未测稳定性**（"稳定 vs 易被破坏"留 v0.3）。
- 仍 n=1、手工语料、构造偏误（措辞门槛随 §2.3 方向而定，预先决定可触发性）。
- `FREQUENCY_CORE_THRESHOLD` 是新自由度，与 persistence 的 `LOCAL_REORG_THRESHOLD=2` 不可直接换算——阈值耦合问题在第二道闸上重现，须做离线 sweep（复用 [`tools/threshold_sweep.py`](../../tools/threshold_sweep.py) 思路，扩展到双闸网格）。
- §2.3 的方向选择本身是协议层变更（尤其方向三把 dedup 路径变成 reorg 入口），须在 v4 显式声明并审计。
- **死结在 v0.1 阶段封死了 reorg，但未污染 v0.1 结论**——因 v0.1 reorg 从未触发、共用规则未产生观测效应。此边界为 v0.2 的工作前提，非 v0.1 的缺陷。
