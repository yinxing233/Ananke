# Persistence-Based Multi-Timescale Memory Architecture

## 系统设计文档（MVP v0.1）

---

## 零、理论约束与工程参数的分离

### 底层原则（理论核心，面试/论文用）

- **原则A：** 不同时间尺度的稳定结构，必须拥有不同的更新机制。
- **原则B：** 长期稳定结构的形成，依赖不可控外部输入的持续检验，而不是系统内部的自我循环。

### 不可修改的工程约束（原则A/B的展开，实现检查用）

| 编号 | 约束                                                     | 来源              |
| ---- | -------------------------------------------------------- | ----------------- |
| C1   | 三层结构，沉淀速率分离                                   | 原则A的最小实现   |
| C2   | 跨层迁移由存续检验驱动，而非访问频率驱动                 | 原则B的迁移准则   |
| C3   | 外部验证的权重方向性高于内部激活                         | 原则B的数值化表达 |
| C4   | 快层不能直接回写慢层，每次跨层迁移必须经过对应的存续检验 | 原则A的保护机制   |

### 可修改的工程超参数（实验后可调，不影响理论约束）

cosine阈值（0.6 / 0.85 / 0.9）、外部验证与内部激活的具体权重比（MVP初值1/e）、persistence_score迁移阈值（MVP初值3.0）、快层容量（MVP初值50条）、中层容量（MVP初值200条）、local_reorganization_trigger阈值（MVP初值2）、存储方案（JSONL / chromadb）、sentence-transformers具体模型。

**版本纪律：后续所有新想法默认进入v0.2待办列表，不回头修改v0.1。**

## 一、项目定位

### 要证明的核心命题

长期稳定记忆的本质是沉淀速率分离（rate separation）而非容量扩张。跨层迁移的机制应该是 persistence-based 的存续检验，而非 frequency-based 的访问频率。

### 理论与实验的关系

本实验不直接验证约束场理论整体。实验验证的是约束场理论的一个工程推论：如果长期稳定结构等于通过存续检验逐步沉淀，那么基于存续检验的记忆系统在长序列任务中，应该比频率驱动系统更稳定地保持结构性信息。

类比：自由能原理启发了主动推断系统，某个主动推断机器人实验失败不等于自由能原理被证伪。本实验的成败同理。

### 与现有方案的差异

| 方案               | 分层逻辑                                       | 跨层迁移机制                                    | 本质隐喻           |
| ------------------ | ---------------------------------------------- | ----------------------------------------------- | ------------------ |
| Mem0               | 无分层，向量存储                               | 无迁移，按相似度检索                            | 搜索引擎           |
| Zep/Graphiti       | 时序知识图谱                                   | 显式图更新                                      | 版本管理           |
| Letta/MemGPT       | core/recall/archival 三层                      | 基于访问频率和容量                              | 操作系统内存管理   |
| RecMem             | Subconscious/Episodic/Semantic 三层            | 基于语义集群的重现频次（recurrence）触发LLM总结 | 频率驱动的延迟压缩 |
| Redis Agent Memory | Working/Long-term + Aging                      | 基于生命周期管理（Promotion/Forgetting）        | 缓存管理           |
| SimpleMem          | Semantic Compression + Recursive Consolidation | 默认所有输入都值得语义处理                      | 全量压缩           |
| **本项目**         | 快/中/慢 三层，沉淀速率分离                    | 基于存续检验（persistence test）                | **生物记忆巩固**   |

核心差异点：Letta 的信息从 recall 移到 archival 是因为"不够常用了"；本系统的信息从快层迁移到中层是因为"经受住了反复检验"，从中层迁移到慢层是因为"触发了已有知识的局部重组"。RecMem 的 recurrence 机制本质是语义集群粒度的频率驱动——当某主题被反复提及达到阈值才触发巩固，价值判断的底层逻辑仍是"出现得多 = 值得记住"。本系统的 persistence_score 区分了外部验证和内部激活（一条信息即使只出现两次，若两次都来自独立外部验证，即可达到迁移阈值），且中层具有局部重组检测能力（合并/矛盾），RecMem 采用"合并优先"策略但无矛盾检测，矛盾信息会被静默覆盖。迁入慢层需要证明该记忆改变了已有知识结构，而非仅仅重复出现。

### 工程策略

从零搭建独立的三层记忆系统，不依赖任何现有框架的源码。总代码量预估 500-800 行，大模型辅助编码可行。参考 RecMem 的架构设计思路（三层解耦、轻量向量拦截），但核心迁移逻辑完全自研。通过配置开关在同一套代码中切换实验组（persistence-based）和对照组（frequency-based）两种模式。

### 评测策略

MVP 阶段只验证 persistence-based 系统本身能否正确运行：记忆写入、激活检测、存续得分累积、跨层迁移、局部重组检测、全链路日志记录。验证方式为人工构造的小规模测试用例（10-20 轮对话），覆盖以下关键路径：

- 外部验证触发存续得分累积 → 快→中迁移
- 内部激活触发存续得分累积 → 快→中迁移
- 中层局部重组检测（合并 / 矛盾）→ 中→慢迁移
- 快层容量满时的淘汰行为
- 全链路日志的完整性和可读性

公开基准评测（LoCoMo、LongMemEval-S）和 frequency-based 对照实验属于 MVP 完成后的扩展阶段。

## 二、三层记忆架构

### 2.1 层级定义

| 层级 | 名称                            | 沉淀速率 | 更新频率             | 容量限制（MVP） | 淘汰机制（MVP）            |
| ---- | ------------------------------- | -------- | -------------------- | --------------- | -------------------------- |
| 快层 | 工作记忆（Working Memory）      | 最快     | 每次交互可写入       | 50 条           | 容量满时淘汰存续得分最低者 |
| 中层 | 巩固记忆（Consolidated Memory） | 中等     | 经存续检验后迁入     | 200 条          | MVP 阶段不设淘汰           |
| 慢层 | 核心记忆（Core Memory）         | 最慢     | 经局部重组检测后迁入 | 无上限          | MVP 阶段不设淘汰           |

### 2.2 层间关系：单向阀机制

信息流向：慢层 → 中层 → 快层（塑形方向，慢层内容影响快层的推理）。

迁移方向：快层 → 中层 → 慢层（沉淀方向，通过存续检验逐级晋升）。

核心约束（C4）：**快层不能直接回写慢层。** 每次跨层迁移必须经过对应的存续检验。这保证了快层的高频扰动不会破坏慢层的长期积累。

## 三、记忆数据结构

每条记忆（Memory Entry）包含以下字段：

```
MemoryEntry:
  id:                          string     # 唯一标识
  content:                     string     # 记忆内容（自然语言）
  layer:                       enum       # WORKING / CONSOLIDATED / CORE
  created_at:                  timestamp  # 写入时间
  last_activated_at:           timestamp  # 最后一次被激活的时间
  internal_activation:         int        # 内部激活计数（系统推理中调用）
  external_validation:         int        # 外部验证计数（用户/环境再次提供一致信息）
  local_reorganization_trigger: int       # 局部重组触发计数（仅中层使用）
  source_ids:                  list       # 来源记忆的 id（记录迁移路径）
  tags:                        list       # 语义标签（用于检索）
```

### 预留字段（MVP 不实现，未来扩展用）

```
  decay_coefficient:      float      # 衰减系数 λ（未来实现被动衰减时使用）
  persistence_score_log:  list       # 存续得分随时间的变化记录（未来分析用）
```

时间戳字段（created_at、last_activated_at）在 MVP 阶段写入但不参与计算。未来增加被动衰减时，只需新增一个定期扫描函数，用当前时间与 last_activated_at 的差值乘以 decay_coefficient 扣减存续得分，无需修改数据结构。

### 字段命名说明

local_reorganization_trigger（而非 structural_trigger）：MVP 中的局部重组检测基于语义相似度触发的合并/矛盾判断，捕捉的是"局部语义重叠引发的知识重组"，而非完整的"因果网络中的结构影响"。未来版本中，真正的结构影响应通过反事实重要性（counterfactual importance）来衡量——即删除该记忆后多少推理路径失效。当前命名诚实地反映了 MVP 的近似程度。

## 四、存续检验判据

### 4.1 快层 → 中层：重复激活型存续检验

**存续得分计算：**

```
persistence_score = external_validation × 1.0 + internal_activation × (1/e)
```

其中 1/e ≈ 0.368。

含义：外部验证的权重方向性高于内部激活（约束C3）。一次外部验证约等于 2.72 次内部激活。外部信号（用户或环境再次提供一致信息）比内部自我引用更有价值，但内部激活不是无意义的。

**超参数说明：** 1/e 是启发式初值，不具有理论唯一性。理论约束仅要求"外部验证权重高于内部激活"（方向性），具体比值可根据实验结果调整。如果实验发现最优比值为 1:0.5 或 1:0.2，理论不受影响；如果发现内部激活权重应高于外部验证（比值反转），则需要在理论层面回应。

**迁移条件：**

```
当 persistence_score ≥ 3.0 时，该记忆从快层迁移至中层。
```

阈值 3.0 的含义：大约等价于"3 次外部验证"或"约 8 次纯内部激活"或某种中间组合。此阈值为工程超参数，可调。

**快层淘汰机制：**

快层容量上限 50 条。新记忆写入时，若已满，淘汰 persistence_score 最低的一条。

### 4.2 中层 → 慢层：局部重组型存续检验

**核心判据：该记忆是否触发了中层已有知识的局部重组，而不仅仅是在已有结构内增加了一个节点。**

MVP 阶段的操作化定义——局部重组操作包括两种：

| 操作类型             | 定义                                                     | 示例                                         |
| -------------------- | -------------------------------------------------------- | -------------------------------------------- |
| 合并（Merge）        | 两条或多条中层记忆被识别为同一概念的不同表述，合并为一条 | "用户喜欢红色" + "用户偏好暖色调" → 合并     |
| 矛盾标记（Conflict） | 新信息与已有中层记忆产生矛盾                             | "用户之前说喜欢红色" vs "用户现在说喜欢蓝色" |

**MVP 阶段不实现的操作（进入v0.2待办）：**

- 拆分（Split）：LLM 判断准确率难以保证，且不影响核心命题验证。
- 重新分类（Reclassify）：同上。

**迁移条件：**

```
当一条中层记忆的 local_reorganization_trigger ≥ 2 时，该记忆成为慢层候选。
```

每次中层发生合并或矛盾标记操作时，系统记录"这次操作是由哪条记忆的激活触发的"，对应记忆的 local_reorganization_trigger 计数 +1。

阈值 2 的含义：这条信息至少两次导致了中层已有知识的局部重组，说明它不是局部数据点，而是对知识结构有持续影响的信息。

**MVP 近似的局限性声明：** 当前检测基于语义相似度（cosine ≥ 0.9）触发 LLM 判断，捕捉的是局部语义重叠引发的重组。无法捕捉因果链中的关键节点（如：A拥有X，X属于B，B能开启C——其中X不与A或C语义相似，但在因果网络中占据关键位置）。未来版本应走向反事实重要性检测。

### 4.3 慢层管理

MVP 阶段慢层不设淘汰机制。进入慢层的记忆长期保留。慢层记忆在推理时被优先检索（优先级：慢层 > 中层 > 快层），但不主动触发任何下层操作。

未来方向：如果慢层也需要管理，对应的是约束场理论中的"跃迁循环"——慢层约束场本身被摧毁的情况。这是远期问题，MVP 不处理。

## 五、激活检测逻辑

### 5.1 内部激活的检测

每次系统进行推理时，将用户输入与所有快层/中层记忆做语义匹配（向量 cosine similarity）。匹配度超过阈值的记忆被标记为"本次推理中被激活"，internal_activation 计数 +1，last_activated_at 更新。

### 5.2 外部验证的检测

当用户的新输入与某条已有记忆在语义上高度一致（cosine similarity ≥ 0.85），且该信息不是系统主动提问获得的（排除系统引导用户确认的情况），标记为外部验证，external_validation 计数 +1。

两个阈值的区别：内部激活阈值较低（"有关联就算"），外部验证阈值较高（"高度一致才算"）。MVP 阶段先用固定值（内部激活：cosine similarity ≥ 0.6，外部验证：cosine similarity ≥ 0.85），后续可调。两个阈值均为工程超参数。

### 5.3 局部重组的检测

触发条件：**仅在有新记忆从快层迁入中层时触发**，不是每轮交互都触发。

触发后执行以下检查：

1. 新进入中层的记忆与中层所有已有记忆做 cosine similarity 计算
2. cosine similarity ≥ 0.9 的配对 → 进入 LLM 判断
3. LLM prompt 要求只回答"合并""矛盾""无关"三个词之一
4. 确认为合并或矛盾 → 记录触发源记忆的 id，对应记忆 local_reorganization_trigger +1

**LLM 在此处的角色：** 做最窄的语义判断（三选一），不做开放性推理。输出格式固定为单个词，代码端直接解析。

## 六、LLM 的两个使用点

MVP 中 LLM API 调用仅在两个地方发生：

| 用途         | 触发时机                                         | Prompt 要求                            | 输出格式                 |
| ------------ | ------------------------------------------------ | -------------------------------------- | ------------------------ |
| 记忆提取     | 每次用户交互后                                   | 从对话中提取新信息，概括为简短记忆条目 | JSON 列表                |
| 局部重组判断 | 新记忆进入中层时，与 cosine ≥ 0.9 的已有记忆配对 | 判断两条记忆的关系                     | "合并" / "矛盾" / "无关" |

所有其他逻辑（存续得分计算、迁移检查、淘汰、激活计数更新）均为确定性代码，不涉及 LLM。

---

## 七、向量计算方案

语义匹配使用本地 sentence-transformers 模型（如 all-MiniLM-L6-v2，约 80MB），零 API 成本。

MVP 阶段快层 50 条 + 中层 200 条，暴力循环计算 cosine similarity 即可（耗时 < 0.01 秒），无需近似最近邻索引。

存储方案：JSONL 文件或 chromadb，根据辅助编码时的便利性选择。两者在 MVP 规模下无性能差异。

## 八、系统整体流程

```
用户输入
  ↓
语义匹配：与快层、中层所有记忆比对（本地 sentence-transformers）
  ↓
更新激活计数：
  - cosine ≥ 0.6 的快层/中层记忆 → internal_activation +1
  - cosine ≥ 0.85 且非系统引导 → external_validation +1
  ↓
写入新记忆：
  - 调用 LLM 从本次交互中提取新信息，写入快层
  - 如果快层已满，淘汰 persistence_score 最低者
  ↓
快层 → 中层迁移检查：
  - 遍历快层，persistence_score ≥ 3.0 者迁移至中层
  ↓
中层局部重组检查（仅在有新记忆进入中层时触发）：
  - 新记忆与中层已有记忆做 cosine similarity
  - ≥ 0.9 的配对进入 LLM 判断（合并/矛盾/无关）
  - 确认的操作 → 记录触发源，local_reorganization_trigger +1
  ↓
中层 → 慢层迁移检查：
  - 遍历中层，local_reorganization_trigger ≥ 2 者迁移至慢层
  ↓
生成回复：
  - 推理时参考所有三层记忆（慢层 > 中层 > 快层 优先级）
  ↓
输出
```

## 九、全链路操作日志

每次以下事件发生时，写入一行带时间戳的日志：

| 事件类型     | 日志内容                                                                      |
| ------------ | ----------------------------------------------------------------------------- |
| 记忆写入     | 时间、记忆ID、内容摘要、写入层级                                              |
| 内部激活     | 时间、记忆ID、触发输入摘要、当前 internal_activation 值                       |
| 外部验证     | 时间、记忆ID、触发输入摘要、cosine similarity 值、当前 external_validation 值 |
| 快层淘汰     | 时间、被淘汰记忆ID、persistence_score                                         |
| 快→中迁移    | 时间、记忆ID、persistence_score                                               |
| 局部重组判断 | 时间、新记忆ID、配对记忆ID、LLM判断结果（合并/矛盾/无关）                     |
| 中→慢迁移    | 时间、记忆ID、local_reorganization_trigger 值                                 |

日志格式固定，方便后续提取实验数据、绘制记忆生命周期曲线。

## 十、MVP 验证方案

### 验证目标

确认 persistence-based 三层记忆系统的所有机制能正确运行，不涉及与其他系统的对比。

### 验证方式

人工构造小规模测试对话（10-20 轮），设计为覆盖系统的关键路径：

| 测试路径        | 验证内容                                                            | 通过标准                                         |
| --------------- | ------------------------------------------------------------------- | ------------------------------------------------ |
| 外部验证 → 迁移 | 重复提供一致信息，观测 external_validation 累积和快→中迁移          | persistence_score 达到阈值后记忆出现在中层       |
| 内部激活 → 迁移 | 提供与已有记忆相关但不完全一致的输入，观测 internal_activation 累积 | 激活计数正确递增，迁移在预期轮次发生             |
| 局部重组 — 合并 | 向中层引入与已有记忆语义高度重叠的新记忆                            | LLM 返回"合并"，local_reorganization_trigger +1  |
| 局部重组 — 矛盾 | 向中层引入与已有记忆矛盾的新记忆                                    | LLM 返回"矛盾"，local_reorganization_trigger +1  |
| 中→慢迁移       | local_reorganization_trigger 累积到阈值                             | 记忆出现在慢层                                   |
| 快层淘汰        | 快层写满后继续写入                                                  | persistence_score 最低者被淘汰，日志记录淘汰事件 |
| 日志完整性      | 全流程运行后检查日志                                                | 所有事件类型均有记录，字段完整                   |

### 测试用例设计原则

测试对话不需要虚构世界观或复杂剧情，只需要能触发上述路径的最小输入序列。例如：前5轮重复提及同一事实（触发外部验证累积），第6-8轮提及相关但不同的信息（触发内部激活），第10轮引入矛盾信息（触发矛盾检测）。

## 十一、技术栈（MVP）

| 组件         | 选择                           | 原因                                                  |
| ------------ | ------------------------------ | ----------------------------------------------------- |
| 语言         | Python                         | 熟悉度高，大模型辅助编码成功率高                      |
| LLM API      | 待定（DeepSeek/Claude/GPT）    | 按成本和效果选择                                      |
| 向量计算     | sentence-transformers 本地模型 | 零 API 成本，MVP 规模下暴力循环即可                   |
| 存储         | JSONL 或 chromadb              | MVP 阶段按编码便利性选择                              |
| 局部重组判断 | LLM API 调用                   | 合并/矛盾的判断需要语义理解，但限制为最窄的三选一判断 |

## 十二、MVP 阶段的边界

### 做的

- 三层记忆的存储与检索
- 快→中的 persistence-based 迁移（存续得分：外部验证 × 1.0 + 内部激活 × 1/e）
- 中→慢的局部重组检测迁移（合并 + 矛盾两种操作）
- 快层的容量淘汰
- 全链路操作日志
- 小规模测试用例验证系统正确性

### 不做的

- 被动时间衰减（预留字段，不实现）
- 慢层的淘汰机制
- 慢层对中层/快层的主动塑形（MVP 阶段慢层只被动参与推理检索）
- 拆分（Split）和重新分类（Reclassify）操作
- 反事实重要性检测
- 多模态记忆（只处理文本）
- frequency-based 对照实验
- 公开基准评测（LoCoMo / LongMemEval-S）
- 与 RecMem 等现有系统的横向对比

## 十三、开放问题（v0.2 及后续探索）

1. **被动衰减的引入：** 数据结构已预留。引入后需要校准衰减系数 λ——不同层的 λ 应该差多少？

2. **从局部重组到反事实重要性：** MVP 的局部重组检测基于语义相似度触发，未来应走向"删除该记忆后多少推理路径失效"的反事实重要性检测。这是 persistence-based 迁移机制的理论完整版。

3. **存续检验的自修改：** MVP 的阈值是固定的。未来系统是否应该能根据运行经验调整自己的阈值？这对应约束场理论中"选择函数内化"的问题——MVP 实现的是骨架层（存续检验由外部设计者固定），未来版本可能需要实现第一次复杂度跃迁（系统内化存续检验的一部分）。

4. **外部验证信号的丰富化：** MVP 阶段的"外部验证"来自基准数据集中用户侧的自然对话输入。真正的存续检验需要系统暴露在不可预测的反馈中。未来部署场景中，外部验证信号可能来自多种通道（用户反馈、环境状态变化、第三方数据源），如何设计这些通道的接入是工程问题。

5. **1/e 权重的经验验证：** 内部激活 vs 外部验证的权重比（1/e : 1）是启发式初值。理论仅约束方向性（外部 > 内部），具体比值由实验数据决定。

6. **局部重组检测的精度：** LLM 做合并/矛盾判断的准确率如何？误判（假阳性）会导致不该进慢层的信息进入慢层，这对系统长期稳定性有什么影响？

7. **慢层的主动塑形：** MVP 阶段慢层只被动参与推理检索。未来版本中，慢层是否应该主动影响下层结构（如：慢层记忆改变快层/中层的激活阈值）？这对应约束场理论中"慢层约束场塑形快层可能性空间"的机制。

8. **扩展评测路线：** MVP 验证系统正确性后，后续阶段按优先级展开：（1）增加 frequency-based 对照组，同一套代码通过配置开关切换，验证 persistence vs frequency 的核心命题；（2）接入 LoCoMo 基准跑分，与 RecMem 等已发表系统横向对比；（3）可选接入 LongMemEval-S。RecMem 开源代码自带 LoCoMo 评测模块，可复用其评测脚本降低接入成本。
