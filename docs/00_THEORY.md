# 00 · THEORY（理论层 / 宪法）

> 本文件只写**几年不变**的理论公理。实验协议、实现、研究过程分别在 `01_PROTOCOL_v3.md` / `02_IMPLEMENTATION.md` / `03_RESEARCH_LOG.md`。
> 改动本文件 = 改变项目身份，须 PI 明确批准。
> 本项目知识按**可变性分层**（结构自同构于约束场理论），见 §6。

## 0. 项目身份
- Ananke 不是"更好用的记忆系统"产品，而是**约束场理论的 AGI 架构应用实验仪器**。
- 真正的实验对象 = 约束场理论的核心命题，而非记忆系统的工程指标。
- 三层结构 / 检索 / 回复 / 嵌入 / LLM 接口，都只是让理论**可操作化、可观测、可证伪**的脚手架。

## 1. 理论根节点
- 根 = **约束场理论**。权威文本（PI 提供）：
  - `《3.约束场理论：当前版本的完整陈述》`
  - `《5.约束场理论的 AGI 架构应用：推演存档》`
- 原则 A / B 不是根，是约束场理论在记忆架构上的**两条推论**。

## 2. 理论 → Ananke 实现的精确映射
| 约束场理论 | Ananke 实现 |
|---|---|
| 沉淀速率分离（快/中/慢层各自独立更新频率与条件） | 原则 A + 三层结构（Working / Consolidated / Core） |
| 单向阀机制（慢→快塑形，快∤回写慢） | 单向阀迁移（只升不降） |
| 反身性条件（不可被系统自身消除的外部验证）+ "疏而非堵"通道 | 原则 B + 外部验证 ≥ 0.85 且"排除系统引导" |
| 存续检验由环境执行（非设计者指定内容） | 外部验证只接受实验者未控制的新输入 |

## 3. 两条核心推论（原则 A / B）
- **原则 A（不同时间尺度要有不同更新机制）** → 沉淀速率分离 → 三层 + 各自独立升层条件。
- **原则 B（长期稳定结构依赖不可控外部输入的持续检验，而非系统内部自我循环）** → 反身性条件 + 外部验证通道 + "系统自生成内容永不计入外部验证"红线。

## 4. Phase 3 的精确理论含义：Internal Selection vs External Selection
- **External Selection（Persistence 模式）**：迁移由**环境**产生的选择压力驱动（外部验证权重 1.0）→ 预测产生受环境约束的稳定结构。
- **Internal Selection（Frequency 模式）**：迁移由**系统内部**产生的选择压力驱动（仅用内部访问次数，忽略外部验证权重差异）→ 退化为内部自循环 = 约束场理论"RLHF 退化"原型（选择压力可被自身消除）→ 预测产生易被破坏的不稳定结构。
- **比较器**（冻结于协议）：同语料两模式，**唯一变更 = Migration Rule（选择压力来源）**，其余全同。比较对象不是"留下多少记忆"，而是"存活记忆 vs **演化中外部世界**后期发展"的一致性（Adaptive System 框架）。
- **抽象层级**：Phase 3 比较的是**选择压力的来源**，Persistence / Frequency 只是其两种实现；若未来换其它实现仍属 External Selection，理论依旧成立。

## 5. 理论自限（务必守，否则即漂移）
- 约束场理论**不提供优化建议**（不说什么约束场"更好"）。
- 约束场理论**不直接预测具体结构**，只预测**可能性空间形状**（分层 / 外部门控 / 单向阀）。
- → Ananke 的 Evidence 应理解为"系统是否产生了理论预测的结构形状"，而非"记忆内容对不对"或"系统更好用了"。
- 任何朝"优化记忆系统 / 提升回复质量"走的动作，都是越界（见 `03_RESEARCH_LOG.md` 的漂移探测器）。

## 6. 四层知识框架（本项目自身的结构自同构）
- **Theory → Protocol → Implementation → Evidence** 是实验的四层；项目自身知识也按可变性分层：
  - Theory（几年） → `00_THEORY.md`
  - Protocol（数月，每轮实验一版） → `01_PROTOCOL_vN.md`
  - Implementation（数天） → `02_IMPLEMENTATION.md`
  - Research Log（数小时） → `03_RESEARCH_LOG.md`
- **结构自同构**：Ananke 作为约束场理论的仪器，其自身知识组织也应遵循"沉淀速率分离"——把不同变化速率的内容混写，等于在仪器内部违反了它要验证的理论。
- 协议（Protocol）是 Theory 与 Implementation 之间**唯一合法的映射层**：理论不会直接面对世界，世界也不会直接验证理论，中间必须隔着操作定义。

## 7. 三阶段各自证伪哪层映射
- **Phase 1 = Theory → Implementation**：实现是否忠实于理论设计（迁移规则 = Persistence 时，系统是否稳定产生符合预期的动力学轨迹）？**只证明实现没偏离理论，不证明 Persistence 优于 Frequency，更不证明理论正确。**
- **Phase 2 = Implementation → Dynamics**：闭环是否真产生可观测的长期行为（慢层持续塑形快层推理）？
- **Phase 3 = Theory A vs B → Different Dynamics**：Persistence 与 Frequency 是否导致不同演化结果？

## 8. Phase 1 精确可证伪命题
> 若迁移规则 = Persistence，则系统能稳定产生符合该规则预期的动力学轨迹（记忆在累积足够外部验证后升层、未获检验者停留或淘汰、且无任何记忆违反单向阀）。
实现忠实即 Phase 1 完成；若 Evidence 显示实现偏离，那是实现 bug 或协议漏洞，不是理论证伪。
